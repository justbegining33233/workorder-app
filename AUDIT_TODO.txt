NOTE: The largest category of major bugs is the pervasive use of
browser alert() / confirm() / prompt() for user feedback and
confirmations. These are:
  • Blocked by default in many browsers when the page is cross-origin

  • Suppressed entirely on iOS PWA home-screen mode
  • Inaccessible (no ARIA, no keyboard focus management)
  • Visually inconsistent with the rest of the app UI
ALL should be replaced with inline error/success states, toast
notifications, or modal dialogs using existing UI patterns.
