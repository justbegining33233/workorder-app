Prisma Files

Prisma is the ORM layer between FixTray and PostgreSQL. Every database read/write goes through the Prisma client. These files are absolutely critical — the entire app depends on the schema and generated client.
FileWhat It Holds / InternalsUsed?Impact if RemovedWhat Would Break
prisma/schema.prisma The single source of truth for the entire database structure. Defines: the PostgreSQL provider using DATABASE_URL and DATABASE_URL_UNPOOLED env vars (Neon pooled + direct connections). Contains 76 models covering every entity in the system. The prisma generate command reads this file and outputs the TypeScript @prisma/client package that every API route imports. YES CRITICAL Cannot generate Prisma client. All database operations fail. App cannot build or run at all.
prisma/dev.db Legacy SQLite database file from early development. The production schema now uses PostgreSQL. This file is a leftover from when the project used SQLite. It is NOT the active database and should not be committed. NO SAFE Nothing. The app uses PostgreSQL (Neon) in both dev and prod. This file is unused.
prisma/migrations/ Contains Prisma migration files that track the history of schema changes. The 0_init migration contains the initial full schema SQL. Running prisma migrate deploy applies all pending migrations to the target database. Required for production database setup and schema updates. YES MAJOR Cannot run prisma migrate deploy. Production database setup breaks. Existing data is safe but schema updates require manual SQL.
prisma/migrations_sqlite_backup/ Backup of old SQLite migration files from when the project used SQLite as its database. These are archived and no longer run. The production database is PostgreSQL. NO SAFE Nothing. These are historical artifacts only.

All 76 Database Models — Grouped by Domain

Every model in schema.prisma maps directly to a PostgreSQL table. Below is each model, what data it stores, and which part of the app depends on it.

Users & Authentication

Customer

End customers who submit work orders. Holds email, password (bcrypt), name, phone, stripeCustomerId, tags, internalNotes, emailVerified status, and loyalty/reward data.

Tables: customers · Relations: WorkOrder, Vehicle, Notification, PaymentMethod, Review, Appointment, Message, RewardClaim

Shop

Auto repair shop accounts. Holds credentials, contact info, business license, insurance, shop type (diesel/gas/etc), Stripe Connect account, 2FA settings, approval status, and profile completeness flag.

Tables: shops · 40+ relations to all shop-scoped entities

Tech

Technicians and managers employed by a shop. Holds credentials, role (tech/manager), availability, GPS coordinates, hourly rate, pay type, bank account info (encrypted), emergency contact, hire/termination dates.

Tables: techs · Relations: WorkOrder, TimeEntry, Shift, PayStub

Admin

Platform super-administrators. Holds username, email, password, role (superadmin/admin), 2FA secret (encrypted), and active status.

Tables: admins · Relations: AuditLog

RefreshToken

JWT refresh tokens stored server-side. Holds hashed token (bcrypt), userId, role, associated shopId, IP address, user agent, expiry, revocation status, and CSRF token embedded in metadata JSON.

Tables: refresh_tokens · Used by: /api/auth/refresh

VerificationToken

One-time tokens for email verification and password reset. Stores SHA-256 hashed token, userId, type (email_verify/password_reset), expiry, and optional metadata.

Tables: verification_tokens

Permission

Per-tech granular permission overrides. Stores shopId, techId, permission key (e.g. workorders.create), and boolean granted/denied value. Overrides the default role-based permissions in lib/permissions.ts.

Tables: permissions

Work Orders & Vehicle Services

WorkOrder

The core entity of the entire app. Holds service location (roadside/in-shop), vehicle type, service details (JSON arrays: repairs, maintenance), parts info, issue description, symptoms, photos (JSON array of URLs), vehicle location (GPS or address), status (pending → assigned → in-progress → closed), assigned tech, estimated cost, amount paid, Stripe payment intent ID, DTC codes, completion date, customer/shop/vehicle references.

Tables: workorders · Most-queried table in the app

StatusHistory

Immutable audit trail of every status change on a work order. Records old status, new status, who changed it (tech/shop/customer), and timestamp. Enables full work order history timeline.

Tables: status_history

Vehicle

Customer-owned vehicles. Stores vehicle type, make, model, year, VIN, license plate, mileage, and fleet account association. Linked to Customer and WorkOrder.

Tables: vehicles

WorkAuthorization

Digital repair authorizations sent to customers. Holds work order reference, authorization status (pending/approved/declined), authorized amount, signature data, and sent/signed timestamps.

Tables: work_authorizations

RecurringWorkOrder

Templates for scheduled recurring work orders (e.g. oil change every 3 months). Holds frequency, next due date, service template data, and customer/shop references.

Tables: recurring_work_orders

WorkOrderTemplate

Shop-specific work order templates to speed up common job creation. Stores service type, description, repairs/maintenance arrays, estimated cost, and labor hours.

Tables: work_order_templates

Scheduling & Appointments

Appointment

Customer-scheduled service appointments. Holds scheduled date/time, service type, vehicle, bay assignment, status (scheduled/confirmed/completed/cancelled), and customer notes.

Tables: appointments

Bay

Physical service bays in a shop. Holds bay name, status (available/occupied/reserved), current work order assignment, and capacity settings.

Tables: bays

ShopSchedule

Shop business hours per day of week. Holds dayOfWeek (0–6), isOpen flag, openTime and closeTime in "HH:MM" format. Used by the appointment booking system.

Tables: shop_schedules

ShopBlockedDate

Dates when a shop is closed (holidays, vacations). Prevents appointment booking on those dates. Stores date and reason.

Tables: shop_blocked_dates

LoanerVehicle

Loaner vehicles a shop can offer to customers. Holds vehicle details, availability status, current customer assignment, and check-in/out timestamps.

Tables: loaner_vehicles

Inventory & Purchasing

InventoryItem

Parts and supplies in a shop's inventory. Holds part name, SKU, part number, category, quantity on hand, reorder threshold, cost, sale price, vendor info, and location (bin/shelf). Core to the inventory management feature.

Tables: inventory_items

InventoryRequest

Tech requests for inventory items. Holds item name, quantity, urgency level, status (pending/approved/denied), requesting tech, and manager response notes.

Tables: inventory_requests

InventoryStock

Stock adjustment records. Tracks quantity changes (receive, issue, return, adjustment) with reason codes and before/after quantities. Audit trail for inventory movements.

Tables: inventory_stock

PurchaseOrder

Purchase orders to vendors for parts/supplies. Holds vendor info, order status (draft/submitted/received), total amount, expected delivery date, and line items.

Tables: purchase_orders

PurchaseOrderItem

Individual line items on a purchase order. Holds part number, description, quantity ordered, unit cost, and received quantity.

Tables: purchase_order_items

CoreReturn

Core charge returns tracking. When a rebuilt part (alternator, starter, etc.) is returned as a "core" for credit, this records the part, core charge amount, return status, and credit applied.

Tables: core_returns

Vendor

Parts suppliers and service vendors for a shop. Holds contact name, phone, email, website, category, account number, payment terms, rating, and active status.

Tables: vendors

Payments & Subscriptions

Subscription

Shop subscription to FixTray platform plans. Holds plan name (starter/growth/professional/business/enterprise), status (active/trialing/past_due/cancelled), Stripe subscription ID, price ID, current period start/end, trial end date, cancel at period end flag.

Tables: subscriptions · Gated by: subscription-limits middleware

PaymentMethod

Stored customer payment methods (Stripe card tokens). Holds Stripe payment method ID, card brand, last 4 digits, expiry, and default flag.

Tables: payment_methods

PaymentHistory

Record of all subscription payments. Holds amount, currency, Stripe charge ID, status, and billing period. Used in the admin billing dashboard.

Tables: payment_history

PaymentLink

One-time payment links sent to customers for work order invoices. Holds Stripe Payment Link ID or custom token, amount, work order reference, expiry, and usage status.

Tables: payment_links

TaxRule

Shop-specific tax configuration. Holds tax name, rate (percentage), applicable service categories, and active status. Applied during invoice generation.

Tables: tax_rules

EnvironmentalFee

Environmental/disposal fees added to invoices (e.g. oil disposal, tire recycling). Holds fee name, amount, applicable service types, and active flag.

Tables: environmental_fees

Communications

Message

In-app messages between shops and customers regarding a work order. Holds sender role, content, work order reference, read status, and delivery timestamp.

Tables: messages

CustomerMessage

Customer portal messages — general messages from customers that may not be tied to a specific work order. Distinct from the work-order-specific Message model.

Tables: customer_messages

DirectMessage

Direct messages between shop staff (tech-to-manager, tech-to-tech). Separate from customer-facing messages. Holds sender ID, recipient ID, content, and read status.

Tables: direct_messages

Notification

In-app notifications for customers. Holds type (work_order_update/payment_due/etc.), title, message, workOrderId reference, read flag, and delivery methods (in-app/email/sms CSV).

Tables: notifications

PortalChatMessage

Internal staff chat messages in the shop/manager/tech portal. Holds role (shop/manager/tech/shared), channelId (global or pair:userId1:userId2), sender name, and message body.

Tables: portal_chat_messages

PushSubscription

Web Push API subscription objects stored for sending browser push notifications. Holds endpoint URL, p256dh key, auth key, and customerId/shopId reference. Used by lib/serverPush.ts.

Tables: push_subscriptions

Payroll & HR

TimeEntry

Clock-in/clock-out records for technicians. Holds clock in time, clock out time, break duration, total hours calculated, work order reference (if clocked to a job), GPS coordinates at clock-in, and approval status.

Tables: time_entries

Shift

Scheduled work shifts for technicians. Holds scheduled start/end times, actual start/end times, shift status, and any notes. Linked to the payroll system.

Tables: shifts

ShiftSwapRequest

Tech-to-tech shift swap requests. Holds requester, target tech, original shift, requested shift, status (pending/approved/denied), and manager approval. Has dual relations to Tech (requester + target).

Tables: shift_swap_requests

LeaveRequest

Time-off requests from technicians. Holds leave type (vacation/sick/personal), start/end dates, status (pending/approved/denied), reason, and manager notes.

Tables: leave_requests

PayPeriod

Defined pay periods for a shop. Holds start date, end date, status (open/closed/paid), and totals. Groups time entries for payroll calculation.

Tables: pay_periods

PayStub

Generated pay stubs for technicians. Holds pay period reference, regular hours, overtime hours, gross pay, deductions, net pay, and payment date. Encrypted bank account info for direct deposit.

Tables: pay_stubs

OvertimeRule

Overtime calculation rules per shop. Defines daily OT threshold (e.g. over 8hrs), weekly OT threshold (e.g. over 40hrs), double-time rules, and multiplier rates.

Tables: overtime_rules

Fleet & Enterprise

FleetAccount

Business fleet accounts (companies with multiple vehicles). Holds company name, contact, credit terms, billing address, credit limit, and current balance. Enables B2B fleet billing.

Tables: fleet_accounts

FleetVehicle

Individual vehicles belonging to a fleet account. Linked to both FleetAccount and Vehicle models.

Tables: fleet_vehicles

FleetInvoice

Consolidated invoices for fleet accounts covering multiple work orders in a billing period. Holds total amount, work order references, due date, and payment status.

Tables: fleet_invoices

Tenant

Enterprise multi-tenant organizations. Holds company name, subdomain (e.g. joes-auto.fixtray.app), logo, contact info, address, plan/status, maxUsers, maxWorkOrders, timezone, currency, and business hours. Enables white-label enterprise deployments.

Tables: tenants

MdmDevice

Mobile Device Management enrolled devices. Holds device ID, platform (iOS/Android), model, OS version, enrollment status, last check-in, and assigned user.

Tables: mdm_devices

MdmPolicy

MDM security policies. Holds policy name, config JSON (password requirements, encryption, app restrictions), and active status.

Tables: mdm_policies

MdmDevicePolicy

Junction table linking MDM devices to applied policies. Tracks enforcement status and last applied timestamp.

Tables: mdm_device_policies

MdmCommand

Remote commands sent to MDM devices (lock, wipe, install app, etc.). Holds command type, payload, status (queued/sent/executed/failed), and response.

Tables: mdm_commands

Inspections & Compliance

DVIInspection

Digital Vehicle Inspections. Holds vehicle reference, inspector tech, inspection date, overall result (pass/fail/advisory), customer approval status, and linked work order. Parent to DVIItem records.

Tables: dvi_inspections

DVIItem

Individual inspection line items within a DVI. Holds system name (brakes, tires, fluids, etc.), condition (good/fair/poor/critical), tech notes, photos array (URLs), and recommended action.

Tables: dvi_items

VehicleConditionReport

Pre-service condition reports documenting a vehicle's state when it arrives. Holds damage notes, photos, mileage, fuel level, and customer signature acknowledgment.

Tables: vehicle_condition_reports

StateInspection

State emissions/safety inspection records. Holds inspection type, result (pass/fail), inspector tech, sticker number, expiry date, and notes. State-compliance record keeping.

Tables: state_inspections

DTCLookup

Diagnostic Trouble Code lookup cache. Stores DTC code (e.g. P0301), description, severity, possible causes, and fix recommendations. Populated by the DTC lookup API and cached here to reduce external API calls.

Tables: dtc_lookup

Reviews, Loyalty & Marketing

Review

Customer reviews of a shop. Holds star rating (1-5), written review, work order reference, public/private flag, shop response, and moderation status.

Tables: reviews

RewardClaim

Loyalty program reward claims. Holds tier ID (tier-1 through tier-4), status (pending/redeemed/expired), claimed and redeemed timestamps, and expiry date. Linked to Customer.

Tables: reward_claims

Referral

Customer referral tracking. Holds referrer customer ID, referred email, referral code, conversion status (pending/converted), and reward credit applied.

Tables: referrals

Campaign

Shop marketing campaigns (email/SMS blast campaigns). Holds campaign name, type, target audience (all customers, fleet, by service type), message template, send schedule, and delivery stats.

Tables: campaigns

FavoriteShop

Customer's bookmarked/favorite shops. Simple junction of Customer → Shop with timestamp. Used in the customer portal "My Shops" view.

Tables: favorite_shops

Shop Configuration & Settings

ShopSettings

Configurable settings per shop. Holds notification preferences (email/SMS toggles), payment settings, tax config, default labor rate, invoice template, appointment settings, and feature toggles.

Tables: shop_settings · One-to-one with Shop

ShopBranding

White-label branding settings per shop. Holds logo URL, primary color, secondary color, custom domain, header/footer text, and "powered by" visibility toggle.

Tables: shop_branding · One-to-one with Shop

ShopLaborRate

Custom labor rate categories for a shop (e.g. Standard Rate, Premium Rate, Diesel Specialist). Each has a name, hourly rate, and category tag.

Tables: shop_labor_rates

ShopService

Services a shop offers. Holds service name, category (diesel/gas), price, duration in minutes, and description. Displayed to customers when creating work orders.

Tables: shop_services

ShopLocation

Multiple physical locations for a shop. Holds location name, address, city, state, zip, phone, email, isMain flag, and status. Enables multi-location shop management.

Tables: shop_locations

IntegrationConfig

Third-party integration credentials and settings per shop. Holds provider name (QuickBooks, Mitchell1, etc.), API keys (encrypted), sync settings, and last sync timestamp.

Tables: integration_configs

Automation & Workflow

AutomationRule

Configurable workflow automation rules. Holds trigger event (status_changed, payment_received, etc.), conditions (JSON), actions (send_email, send_sms, create_notification, etc.), and enabled/disabled status.

Tables: automation_rules

AutomationExecution

Audit log of automation rule executions. Records which rule fired, on which entity, what actions were taken, outcome (success/failure), and execution timestamp.

Tables: automation_executions

ApiKey

API keys for third-party integrations and programmatic access. Holds SHA-256 hashed key (raw key shown once at creation), shop ID, scopes (CSV), label, last used timestamp, expiry, and revocation flag.

Tables: api_keys · Used by: apiKeyAuth.ts

Webhook

Outbound webhook endpoints registered by shops. Holds target URL, HMAC-SHA256 secret, subscribed events (CSV), active flag, and failure count (auto-disables at 10 failures).

Tables: webhooks · Used by: webhookService.ts

Analytics & Logging

ActivityLog

General activity log for shop events (work order created, status changed, payment received, etc.). Holds entity type, entity ID, action, actor info, and timestamp.

Tables: activity_logs

AuditLog

Admin/superadmin audit trail. Every admin action (approve shop, delete user, change settings) is recorded here with admin ID, action type, target entity, IP address, user agent, and shop context.

Tables: audit_logs · Written by: auditLog.ts

PageView

Page view analytics records. Holds path, user ID (if authenticated), session ID, referrer, user agent, and timestamp. Used by the analytics dashboard.

Tables: page_views

TechTracking

GPS location history for technicians. Stores latitude, longitude, accuracy, speed, heading, battery level, and timestamp per location ping. Powers the TechLiveMap component.

Tables: tech_tracking

ShopMonthlyReport

Pre-aggregated monthly performance reports per shop. Holds revenue totals, work order counts, average ticket value, tech productivity metrics, and top services for the reporting period.

Tables: shop_monthly_reports

Media & Documents

Photo

Work order photos stored in Cloudinary. Holds Cloudinary URL, public ID, filename, caption, work order reference, uploader ID, and upload timestamp. Used by the photo capture and gallery features.

Tables: photos · Companion file: data/photos.json (dev fallback)

CustomerDocument

Documents uploaded by or for a customer (insurance cards, vehicle titles, etc.). Holds file URL, filename, document type, upload date, and visibility (customer-only vs shared with shop).

Tables: customer_documents
