Root Configuration Files

These files sit at C:\FixTray\ (the project root). They configure the build system, deployment targets, runtime server, and tooling. Most are read at build time, not bundled into the app itself.
FileWhat It Holds / InternalsUsed?Impact if RemovedWhat Would Break
package.json Project manifest. Declares all npm dependencies (Next.js, Prisma, Stripe, Socket.IO, Twilio, Sentry, Capacitor, bcrypt, jose, zod, recharts, jsPDF, etc.) and devDependencies (TypeScript, Jest, Playwright, ESLint, Tailwind). Contains all npm run scripts: dev, build, start, test, lint, db:push, db:migrate. Also sets "private": true and version 1.0.0. YES CRITICAL Without it, npm install fails and Node.js cannot resolve any package. The entire project dies.
next.config.ts Next.js runtime & build configuration. Key internals: wraps config with Sentry and next-intl plugins; sets security HTTP headers (CSP, HSTS, X-Frame-Options etc.) on all routes; configures images.remotePatterns for Cloudinary; webpack chunk-splitting (vendor, recharts, sentry bundles); marks resend and twilio as external packages; enables typedRoutes, optimizeCss, scrollRestoration; strips console logs in production. YES CRITICAL Build fails. No security headers served. Cloudinary images broken. Sentry and i18n plugins not applied.
tsconfig.json TypeScript compiler configuration. Sets @/ path alias pointing to ./src (used everywhere as @/components/..., @/lib/...). Enables strict mode, jsx: preserve, targets ES2017. Includes next-env.d.ts and all src/** files. YES CRITICAL TypeScript compilation fails. All @/ imports resolve to nothing. App cannot build.
next-env.d.ts Auto-generated by Next.js. Provides TypeScript type references for Next.js image types and other framework types. Should never be edited manually — Next.js regenerates it. YES MAJOR TypeScript loses Next.js type definitions. Type errors throughout. Auto-regenerated on next next dev run.
postcss.config.mjs PostCSS configuration. Loads @tailwindcss/postcss plugin which processes all Tailwind CSS utility classes from globals.css and component JSX files. Required for Tailwind v4. YES CRITICAL CSS build fails. All Tailwind classes produce no styles. App renders unstyled.
eslint.config.mjs ESLint flat-config file. Uses eslint-config-next preset which includes React, hooks, and accessibility rules. Only runs during npm run lint or in VS Code with ESLint extension. Not part of the production build. DEV ONLY SAFE Linting breaks. No runtime impact — next build can still run.
jest.config.cjs Jest unit test configuration. Sets up ts-jest transformer, jest-environment-jsdom for browser-like environment, maps @/ aliases, configures coverage reporters (clover, lcov, json). Used only by npm test. DEV ONLY SAFE Unit tests cannot run. Zero runtime impact.
playwright.config.ts Playwright end-to-end test configuration. Defines browser projects (chromium, firefox, webkit), base URL, test directory pointing to e2e/, screenshot on failure, and CI-specific settings. Used only by npm run test:e2e. DEV ONLY SAFE E2E tests cannot run. Zero runtime impact.
lighthouserc.json Lighthouse CI configuration. Defines performance score thresholds (performance, accessibility, best-practices, SEO) for automated auditing in CI pipelines. Not referenced by any build or runtime code. DEV ONLY SAFE Lighthouse CI audits fail. Zero app impact.
i18n.ts Internationalization configuration for next-intl. Exports a getRequestConfig function that dynamically imports the correct message JSON from messages/{locale}.json (currently en.json and es.json). Loaded by next.config.ts via createNextIntlPlugin('./i18n.ts'). YES MAJOR next-intl plugin crashes. All translation strings (t('key')) throw. App build fails if next-intl components are used.
instrumentation.ts Next.js server instrumentation hook. Runs once when the Node.js server starts. Registers OpenTelemetry (via @vercel/otel) for distributed tracing with service name fixtray-app. Then conditionally loads sentry.server.config (Node.js runtime) or sentry.edge.config (Edge runtime). Also exports onRequestError to capture all request errors in Sentry automatically. YES MINOR Sentry and OTel stop initializing on server. Error tracking and performance monitoring go dark. App still runs fine.
instrumentation-client.ts Client-side Sentry initialization. Configures Sentry browser SDK with session replay (masks text/media), browser tracing, adjustable sample rates (100% dev, 10% prod for traces; 100% error sessions, 5% clean sessions in prod). Also exports onRouterTransitionStart for route change tracking. YES MINOR Client-side error reporting and session replay stop working. No user-facing impact.
sentry.server.config.ts Server-side Sentry configuration. Sets DSN from SENTRY_DSN env var, environment tag, trace sample rate (20% prod / 100% dev), and filters noise errors like ResizeObserver loop limit exceeded. Loaded by instrumentation.ts dynamically at startup. YES MINOR Server-side error tracking stops. App runs normally.
sentry.edge.config.ts Edge runtime Sentry configuration (for Next.js middleware running in the Vercel Edge Network). Minimal config — DSN, environment, sample rate. Loaded by instrumentation.ts when NEXT_RUNTIME === 'edge'. YES MINOR Middleware/Edge route errors go untracked in Sentry. App runs normally.

Deployment Configuration Files

These files tell different hosting platforms how to build and run FixTray. Only one is needed depending on where you deploy — they don't affect each other.
FileWhat It Holds / InternalsUsed?Impact if RemovedWhat Would Break
vercel.json Vercel deployment config. Defines two cron jobs: /api/cron (daily 7AM) and /api/cron/keepalive (every 5 min). Sets extended function timeouts for heavy API routes (payroll: 60s, messages: 30s, shop stats: 30s etc.). Sets NEXT_PUBLIC_ENABLE_REALTIME=false as a default env var. YES MAJOR On Vercel: cron jobs stop running (no daily cleanup/keepalive), heavy API routes may timeout at 10s default. Locally: no impact.
railway.toml Railway.app deployment config. Specifies build command (npm run build), start command (node server.js — the custom server with Socket.IO), health check path (/health), and restart policy (on failure, max 3 retries). Uses Nixpacks builder. YES MAJOR On Railway: deployment fails to configure correctly. Locally: no impact.
render.yaml Render.com deployment configuration (Infrastructure as Code). Defines web service settings for Render hosting as an alternative to Railway/Vercel. OPTIONAL SAFE Only matters if deploying on Render.com. No impact for Vercel or Railway.
Dockerfile Multi-stage Docker build. Stage 1 (builder): installs all deps, runs npm run build. Stage 2 (runner): copies build output, sets NODE_ENV=production, exposes port 3000, runs npm start. Used for containerized deployments (Docker, Kubernetes, etc.). OPTIONAL SAFE Docker builds fail. No impact for Vercel/Railway deploys or local dev.

Runtime Server Files

These are the actual Node.js server files that run in production on Railway or any non-Vercel host.
FileWhat It Holds / InternalsUsed?Impact if RemovedWhat Would Break
server.js Custom Node.js HTTP server combining Next.js + Socket.IO on a single process. Key internals: creates an http.createServer wrapping Next.js's request handler; adds a /health endpoint for Railway/Render load balancer checks; initializes socket.io on path /api/socket with JWT authentication middleware; handles shop/tech/customer rooms; broadcasts work-order-updated, new-message, tech-location-updated, clock-status-changed events; binds to 0.0.0.0:PORT for external traffic. This is the file referenced by railway.toml's start command. YES MAJOR On Railway/self-hosted: cannot start the production server. Real-time Socket.IO features stop working (live work order updates, live messaging, tech location). On Vercel: not used — Vercel uses Next.js's built-in server.
socket-server.js Standalone Socket.IO-only server (no Next.js). Intended to be deployed on a separate service (Railway, Render, Fly.io) while the Next.js app lives on Vercel. Reads ALLOWED_ORIGINS, JWT_SECRET, and PORT env vars. Exposes its own /health endpoint. Handles identical room/event logic as server.js. Dev command: npm run dev:socket. Used when Next.js is on Vercel (which can't run persistent WebSocket servers). SITUATIONAL MINOR If deployed on Vercel: removing this means no standalone socket server available — real-time features require polling fallback. If on Railway with server.js: this file is unused.

Mobile App Configuration

These files are only relevant for building the Android/iOS mobile apps via Capacitor. The web app runs completely without them.
FileWhat It Holds / InternalsUsed?Impact if RemovedWhat Would Break
capacitor.config.ts Capacitor mobile configuration. Defines appId: "com.fixtray.app", appName: "FixTray Pro". In dev mode (CAPACITOR_LOCAL=true): points the WebView at http://10.0.2.2:3000. In production: loads https://fixtray.app/auth/login. Configures Android (backgroundColor: '#020608', user-agent override, debugging disabled), iOS (scheme fixtray://), and all Capacitor plugins: Camera (1920x1440, 85% quality), Geolocation (high accuracy), Push Notifications, Local Notifications, Secure Storage, Biometric Auth. MOBILE ONLY MINOR Android/iOS builds fail. Web app unaffected — runs normally in browsers.
build-apk.bat Windows batch script that automates the Android APK build pipeline: runs npm run build, then npx cap copy android, then enters the android/ directory and runs Gradle assembleRelease. Convenience script for Windows developers building mobile. MOBILE ONLY SAFE Must build APK manually via Gradle. Web app completely unaffected.
android/ Full Android Studio project generated by Capacitor. Contains build.gradle, settings.gradle, gradle.properties, variables.gradle, gradlew, app/ folder with the main Android app module, and gradle/ wrapper. This is the native shell that wraps the FixTray web app in a WebView for Android distribution. MOBILE ONLY SAFE No Android APK can be built. Web app and server completely unaffected.
ios/ iOS Xcode project generated by Capacitor. Contains debug.xcconfig and the App/ folder. This is the native shell that wraps FixTray for iOS App Store distribution. MOBILE ONLY SAFE No iOS IPA can be built. Web app completely unaffected.
